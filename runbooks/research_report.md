# Runbook: выпуск исследовательского отчёта в Drive

## Предусловия
- OAuth consent получен для `drive.file` и `drive.metadata.readonly`.
- Сервисный runtime имеет доступ к токену пользователя.
- Telegram chat id для подтверждения известен.

## Пошаговый сценарий
1. Принять входную задачу (`task_id`, тема, выводы, аудитория).
2. Сформировать текст отчёта в markdown.
3. Создать файл в Google Drive (`research-report-<task_id>.md`).
4. Назначить permission "доступ по ссылке" (reader).
5. Прочитать `webViewLink` и зафиксировать в журнале.
6. Отправить подтверждение в Telegram со ссылкой.
7. Вернуть `file_id` и ссылку вызывающему процессу.

## Обработка ошибок

### Ошибки доступа
- **Симптомы:** 401/403, `auth_failed`, `insufficientPermissions`.
- **Действия:**
  1. Остановить пайплайн (без повторов).
  2. Пометить задачу как `blocked_auth`.
  3. Запросить у пользователя re-consent OAuth.
  4. После обновления токена перезапустить с шага 3.

### Ошибки квот
- **Симптомы:** 429, `quota_exceeded`, `rateLimitExceeded`.
- **Действия:**
  1. Включить exponential backoff (например 1s, 2s, 4s, 8s).
  2. Максимум 4 повтора для idempotent-операций.
  3. Если исчерпано — статус `delayed_quota`, уведомить оператора.

### Сетевая нестабильность
- **Симптомы:** timeout, `transient_network_error`.
- **Действия:**
  1. Повторить запрос до 3 раз.
  2. Использовать корреляционный `request_id` для трассировки.

### Невозможно выдать публичную ссылку
- **Симптомы:** `permission_denied` на `anyoneWithLink`.
- **Действия:**
  1. Сохранить файл приватным.
  2. Отправить в Telegram сообщение о частичном успехе.
  3. Эскалировать владельцу доменной политики Drive.
