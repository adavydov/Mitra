# CAP-DRIVE-01 — Интеграция с Google Drive

## Назначение
Возможность Митры создавать отчёты как файлы в Google Drive Пользователя и возвращать ссылку для совместного доступа.

## Минимально необходимые OAuth scopes
Для сценария «создать файл отчёта и выдать ссылку» фиксируются следующие scopes:

1. `https://www.googleapis.com/auth/drive.file`
   - доступ только к файлам, созданным/открытым приложением;
   - достаточно для создания нового артефакта и управления его доступом.
2. `https://www.googleapis.com/auth/drive.metadata.readonly`
   - чтение метаданных файла (id, webViewLink, permissions) для валидации публикации ссылки.

> Принцип минимальных привилегий: не использовать полно-дисковый `drive`, пока нет отдельного обоснования.

## Ограничения и безопасность
- Токены хранятся в защищённом хранилище секретов.
- Доступ `anyoneWithLink` включается только для артефактов, где это явно разрешено политикой задачи.
- При ошибках авторизации требуется ре-консент пользователя.
